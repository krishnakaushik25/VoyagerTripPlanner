Steps to run the application on Docker :

Now, I'll provide you with step-by-step instructions to run this application using Docker Desktop:

### Step-by-Step Instructions to Run the Application

1. **Prerequisites**:
   - Make sure Docker Desktop is installed and running on your system
   - Ensure all project files are in the same directory

2. **Build and Start the Containers**:
   Open a terminal in your project directory and run:
   ```bash
   docker-compose up --build
   ```
   This command will:
   - Build both Docker images
   - Create and start both containers
   - Set up the network between them
   - Mount the logs directory

3. **Access the Applications**:
   - Geoclip API will be available at: `http://localhost:8000`
   - Chatbot application will be available at: `http://localhost:8501`

4. **View Logs**:
   - You can view the logs in real-time in the terminal where you ran docker-compose
   - Logs are also persisted in the `./logs` directory on your host machine

5. **Stop the Application**:
   - Press `Ctrl+C` in the terminal where docker-compose is running
   - Or run `docker-compose down` in a new terminal

6. **Additional Commands**:
   - To stop and remove containers: `docker-compose down`
   - To view running containers: `docker ps`
   - To view logs of a specific service: `docker-compose logs -f [service-name]`
     (e.g., `docker-compose logs -f geoclip-api` or `docker-compose logs -f chatbot`)

### Important Notes:
1. The Geoclip API runs on port 8000 and the Chatbot on port 8501
2. The logs directory is mounted as a volume, so logs will persist even if containers are removed
3. The chatbot service depends on the API service, so it will wait for the API to be ready before starting
4. Both services are configured to restart automatically unless explicitly stopped
